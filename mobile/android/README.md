# Click Assistant Android 端

本文档用于说明 `mobile/android/` Android 工程的现状、能力边界与本地运行方式。

## 定位

Android 端迁移 Windows 桌面端中适合手机触控环境的任务管理、步骤编辑、执行控制与日志能力，不提供键盘按键和组合键注入；点击与文本输入继续保留，并针对触控交互增加滑动手势与坐标拾取。当前仍采用依赖较少的 Android 原生实现，降低真机验证成本：

- Java。
- Android framework View（未使用 Jetpack Compose）。
- `SharedPreferences` 以 JSON 持久化多个任务与执行日志。
- `AccessibilityService` 分发手势执行任务。

后续产品化阶段可迁移到 Kotlin、Jetpack Compose 和 Room（见 `docs/planning/MOBILE_EXTENSION_PLAN.md`）。

## 已实现能力（v0.4.0）

- 简化多页面界面：首页只保留“新建任务 / 任务库 / 执行日志”，任务配置与日志分别进入独立页面。
- 多任务库：任务的创建、保存、复制、删除、启用/禁用与切换。
- 常用任务模板：新建时可直接选择点击任务、文本输入任务、点击与文本组合任务或空白任务。
- 点击与文本组合：组合模板按顺序创建“点击目标输入框”和“输入文本”两个步骤，仍可继续增删和排序。
- 多步骤编辑：每个任务可包含多个步骤，支持上移、下移与删除。
- 四种步骤动作类型，对应桌面端动作模型的移动端子集：
  - 点击：坐标、连点次数、步骤后延迟。
  - 滑动：起点 / 终点坐标、滑动持续时间。
  - 等待：等待时长。
  - 文本输入：文本内容、逐字间隔（写入当前聚焦的输入框）。
- 步骤前后延迟：每个步骤可配置执行前与执行后等待。
- 执行前确认：启动前弹出任务摘要，展示重复次数与步骤列表。
- 执行日志：本地保存最近 100 条执行记录，包含状态、原因与时间。
- 执行控制：启动、暂停、继续、停止，对应桌面端执行引擎状态机。
- 悬浮控制窗：任务切到其他应用执行后，仍可展开/收起并执行返回应用、暂停、继续和停止。
- 执行快照：启动时复制任务配置，本次运行不受后续编辑影响。
- 光标坐标拾取：透明取点层显示可拖动的蓝色十字光标，点击步骤确认一次位置，滑动步骤依次确认起点和终点。
- 文本输入失败处理：找不到已聚焦输入框或目标拒绝写入时立即标记失败并停止，避免误报完成。
- 旧版单任务数据自动迁移到多任务结构。

## 安全边界

- 所有任务必须由用户主动配置、主动启动、可随时停止。
- 不提供绕过验证码、登录、支付确认、权限弹窗或系统安全机制的功能。
- 不提供隐藏执行、无感后台执行或诱导用户开启权限的能力。
- 默认不上传任务配置、坐标、文本内容和执行日志。
- 文本输入依赖目标界面存在已聚焦的输入框，否则会记录失败原因并停止任务。

## 本地运行

本机已安装 Android SDK 与 Android Studio（自带 JDK 17）。在工程目录下执行：

```powershell
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
cd mobile/android
.\gradlew.bat assembleDebug
```

生成的 debug APK 位于：

```text
mobile/android/app/build/outputs/apk/debug/app-debug.apk
```

若 Gradle Wrapper 不存在，可先执行 `.\gradlew.bat wrapper` 生成。

## 真机验证

1. 将 debug APK 安装到 Android 真机（最低 Android 7.0 / API 24）。
2. 打开应用，点击“打开辅助功能设置”。
3. 在系统设置中启用“Click Assistant 移动端”。
4. 回到应用，从“新建任务”选择点击、文本输入、点击与文本组合或空白模板。
5. 编辑点击步骤时点击“点选点击位置”，在覆盖层中拖动蓝色光标并确认坐标。
6. 点击“保存任务”，再点击“开始执行”，确认弹窗后切换到安全的目标界面。
7. 运行中使用跨应用悬浮控制窗执行“暂停 / 继续 / 停止”，也可点击“应用”返回主界面。
8. 在“执行日志”中查看最近的执行结果。

涉及真实点击与文本输入时，应先在安全界面中验证，例如空白桌面或测试页面。

## 已知限制

- 坐标基于屏幕像素，未做分辨率、方向与多窗口适配。
- 文本输入仅作用于当前聚焦的输入框，不支持全局键盘注入。
- 不提供键盘按键和组合键步骤；这是移动端与 Windows 端的明确平台差异。
- 暂停发生在步骤之间，无法中断正在进行的单次手势。
- 当前未在 CI 中自动生成移动端 APK，需在本地执行上述 Gradle 命令。
