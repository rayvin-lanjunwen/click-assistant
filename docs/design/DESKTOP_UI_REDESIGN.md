# 桌面端界面改版设计

本文档记录 Click Assistant 桌面端从单页参数面板升级为多页面任务中心的首版设计。

## 1. 设计目标

- 首页只承担新建任务、任务库和执行日志三个入口，不承载复杂编辑。
- 用户先选择任务类型，再进入对应任务编辑页。
- 已保存任务统一进入任务库管理，支持执行、编辑、改名、复制和删除。
- 任务执行时提供基础悬浮窗和点击位置提示，减少用户在目标软件和主窗口之间来回切换。
- 所有界面命名遵循统一术语，避免“任务”“流程”“脚本”“日志”等概念混用。

## 2. 页面结构

桌面端首版采用以下页面：

- 首页：默认首屏，仅展示新建任务、任务库和执行日志三个主入口，以及当前执行状态。
- 新建任务类型选择页：展示鼠标点击、键盘按键、文本输入、组合任务四类入口。
- 任务库页：展示已保存任务，支持编辑基础信息、配置步骤、执行、复制和删除。
- 执行日志页：展示最近执行结果和本次运行记录。
- 鼠标点击任务编辑页：配置点击位置、点击次数、开始延迟、点击间隔，并支持保存和执行。
- 基础悬浮窗：展示运行中、已暂停状态，并提供暂停、继续、停止和展开/收起。

## 3. 首页

首页用于回答三个问题：

- 用户是否要新建任务。
- 用户是否要打开任务库。
- 用户是否要查看执行日志。

首页结构：

```text
Click Assistant
点击与输入助手

[新建任务]
[任务库]
[执行日志]

当前状态 / 停止快捷键
```

## 4. 新建任务类型选择页

任务类型统一为：

- 鼠标点击：固定位置点击、重复点击、连点操作。
- 键盘按键：重复按键、方向键、功能键等。
- 文本输入：自动输入固定文本或长文本。
- 组合任务：多个点击、按键、文本输入和等待步骤组合执行。

首版先完整实现鼠标点击任务编辑页，其他类型保留入口和默认任务创建能力，后续逐步细化专门编辑页。

## 5. 任务库页

任务库用于管理已保存任务。

每个任务至少展示：

- 任务名称。
- 任务说明。
- 步骤数量。
- 是否启用。
- 最近执行状态。

任务操作：

- 执行。
- 编辑基础信息。
- 编辑步骤参数。
- 改名。
- 复制。
- 删除，任务库列表区提供“删除选中任务”入口，避免隐藏在深层确认页。

任务库右侧采用三段式标签页，避免把所有输入项同时暴露给用户：

- 基础信息：只编辑任务名称、重复次数、开始延迟和任务说明。
- 步骤设置：集中管理步骤列表、动作类型和当前步骤参数。
- 执行确认：集中展示执行摘要、安全提醒、停止快捷键和执行/复制/删除等高影响操作。

步骤配置：

- 鼠标点击：编辑 X 坐标、Y 坐标和点击方式，并支持捕获坐标和测试点击。
- 键盘按键：编辑目标按键、连按次数和连按间隔；新建键盘按键步骤默认显示“未设置”，不得自动填入 `A` 或其他目标键；步骤列表摘要必须直接显示目标按键、连按次数和间隔，让用户知道该步骤会持续按哪个键；点击按键捕获框后，可以直接按键盘上的键完成捕获，界面会以大号文本展示当前按键，并显示正在监听、已捕获或暂不支持的状态。
- 组合键：编辑类似 `Ctrl+C`、`Ctrl+Shift+Esc` 的组合键文本。
- 文本输入：编辑要输入的文本内容和字符间隔。

## 6. 执行日志页

执行日志页用于集中展示执行反馈。

页面结构：

```text
执行日志

最近执行结果
任务开始时间 / 结束时间 / 状态 / 消息

本次运行记录
启动、暂停、继续、停止、错误等实时记录
```

## 7. 鼠标点击任务编辑页

鼠标点击任务编辑页采用分组表单，避免直接暴露一排无说明数字。

```text
任务名称

点击位置
X 坐标 / Y 坐标
[获取当前鼠标位置] [测试点击一次]

点击设置
点击次数
开始延迟
点击间隔

执行摘要
开始后等待 N 秒，在 X/Y 坐标点击 N 次，每次间隔 N 秒。

[保存任务] [执行任务]
```

字段说明：

- X 坐标：屏幕左上角为 0，向右增大。
- Y 坐标：屏幕左上角为 0，向下增大。
- 点击次数：完整任务执行时重复点击的次数。
- 开始延迟：点执行后，真正开始点击前等待多久。
- 点击间隔：一次点击完成后，等待多久再执行下一步或下一次循环。
- 点击位置提示：每次真实鼠标点击前，在目标坐标显示虚拟鼠标箭头和扩散动效，帮助确认点击位置。

## 8. 执行中与暂停编辑

执行过程中，用户可以暂停任务。首版约定：

- 暂停后允许回到编辑页调整并保存任务配置。
- 已经启动的本次运行仍以启动时读取的配置为准。
- 如需让新配置立即生效，用户应停止当前任务后重新执行。

后续如果执行引擎支持运行中配置热更新，再扩展为“保存到本次运行”。

## 9. 基础悬浮窗

悬浮窗用于用户切到其他软件时仍能控制任务。

迷你态：

```text
Click Assistant | 运行中 | [暂停] [停止] [展开]
```

展开态：

```text
任务名称
状态：运行中 / 已暂停
位置：X/Y
点击次数与间隔

[暂停] [继续] [停止]
[收起]
```

设计原则：

- 停止按钮始终可见。
- 窗口置顶并可拖动，主窗口最小化后仍保持悬浮。
- 支持展开和收起，避免遮挡目标软件。
- 暂停态显示“继续”和“停止”，并引导用户回主窗口编辑。

## 10. 统一命名

- 软件名称：Click Assistant。
- 中文说明：点击与输入助手。
- 首页入口：新建任务、任务库、执行日志。
- 执行状态：未运行、准备执行、运行中、已暂停、已停止、已完成、执行失败。
- 执行历史统一称为执行日志或运行记录，不使用英文 `log` 作为界面文案。
- 保存好的任务统一称为任务库，不使用“已有保存任务”作为正式入口名。
