# Click Assistant UI/UX 设计

文档版本：v0.1.0
最后更新：2026-07-09
维护人：项目负责人
当前状态：初稿，待确认

本文档用于记录 Click Assistant 首版 MVP 的界面风格、交互原则、页面结构和关键状态设计。

## 1. 设计定位

Click Assistant 是一个工具型桌面软件，界面应优雅、简洁、克制，重点服务于重复操作配置和任务执行控制。

视觉方向：

- 优雅简洁。
- 透明毛玻璃感。
- 信息清晰。
- 操作高效。
- 运行状态醒目。

## 2. 视觉风格

### 2.1 整体观感

- 主窗口使用轻量半透明背景。
- 内容区域使用浅色玻璃面板，边界清晰但不过度厚重。
- 背景可使用柔和的深浅渐变或系统亚克力效果，但不能影响文字可读性。
- 避免过度装饰，所有视觉效果都应服务于信息分组和状态识别。

### 2.2 色彩建议

- 主背景：冷灰、雾白或浅蓝灰。
- 主色：稳重蓝色，用于主要按钮和选中状态。
- 成功状态：绿色。
- 警告状态：琥珀色。
- 危险状态：红色，用于立即停止和删除。
- 文本：深灰或近黑色，保证透明背景下仍有足够对比度。

### 2.3 字体与间距

- 默认字体使用系统字体，例如 Microsoft YaHei UI。
- 正文大小建议 13 到 14 像素。
- 标题大小建议 16 到 20 像素。
- 面板内边距建议 16 像素。
- 控件之间保持稳定间距，避免界面拥挤。

## 3. 主界面布局

主界面采用三栏结构：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶栏：应用名称 / 当前状态 / 设置入口                         │
├──────────────┬──────────────────────────┬──────────────────┤
│ 左栏任务列表  │ 中栏任务详情与步骤配置     │ 右栏运行控制与日志 │
└──────────────┴──────────────────────────┴──────────────────┘
```

### 3.1 左栏：任务列表

内容：

- 新建任务按钮。
- 任务搜索或筛选入口，首版可暂不实现搜索。
- 任务列表。
- 复制和删除按钮。

交互：

- 点击任务后，中栏展示任务详情。
- 删除任务需要二次确认。
- 当前任务应有清晰选中状态。

### 3.2 中栏：任务详情

内容：

- 任务名称。
- 任务描述。
- 重复次数。
- 开始延迟。
- 输入步骤表格。
- 添加步骤、采集坐标、保存任务。

交互：

- 任务运行中禁止编辑当前任务。
- 未保存变更离开时，应提示是否保存。
- 步骤列表支持上移和下移调整顺序。

### 3.3 右栏：运行控制

内容：

- 当前执行状态。
- 当前步骤。
- 已执行轮次。
- 启动、暂停、继续、立即停止按钮。
- 最近日志。

交互：

- “立即停止”按钮必须常驻并明显。
- 运行中突出显示当前状态。
- 停止任务后保留最近日志，方便用户确认结果。

## 4. 关键页面

### 4.1 新建或编辑任务弹窗

用途：

- 配置任务基础信息。
- 配置重复次数和开始延迟。
- 配置默认步骤间隔。

设计要求：

- 弹窗内容保持简洁，不承载复杂步骤编辑。
- 保存按钮只在表单合法时可用。
- 错误提示显示在字段附近。

### 4.2 输入步骤编辑弹窗

用途：

- 编辑单个输入步骤。

设计要求：

- 坐标输入区必须清晰展示 X 和 Y。
- “采集坐标”按钮应靠近坐标输入区。
- 点击类型使用下拉选择。
- 执行前等待和执行后间隔使用数字输入。

### 4.3 坐标采集模式

用途：

- 采集当前鼠标屏幕绝对坐标。

设计要求：

- 显示当前坐标。
- 提示首版仅支持主显示器场景。
- 提供确认和取消。
- 采集模式不应遮挡用户确认目标位置。

### 4.4 设置页面

用途：

- 管理默认倒计时、默认间隔和快捷键。

设计要求：

- 快捷键冲突时，应给出明确提示。
- 立即停止快捷键应有醒目说明。
- 设置保存后应给出轻量反馈。

## 5. 状态设计

### 5.1 空状态

任务列表为空时：

- 显示“还没有点击任务”。
- 提供“新建任务”按钮。

### 5.2 未运行

- 状态标签显示“未运行”。
- 启动按钮可用。
- 暂停、继续按钮不可用。
- 停止按钮可见但不可用或弱化。

### 5.3 运行中

- 状态标签显示“运行中”。
- 启动按钮不可用。
- 暂停和立即停止可用。
- 当前步骤和轮次实时更新。

### 5.4 已暂停

- 状态标签显示“已暂停”。
- 继续和立即停止可用。
- 暂停按钮不可用。

### 5.5 执行失败

- 状态标签显示“执行失败”。
- 展示失败原因。
- 用户可以重新启动任务。

## 6. 交互约束

- 首版使用屏幕绝对坐标。
- 首版不支持多显示器专项适配。
- 首版不提供任务导入导出。
- 任务运行中禁止编辑当前任务，避免配置变更影响执行。
- 删除任务必须二次确认。
- 立即停止必须优先于普通操作。

## 7. 毛玻璃效果约束

- 毛玻璃效果不能牺牲文字可读性。
- 面板透明度不宜过高，避免背景干扰内容。
- 运行控制区和危险操作按钮不应使用低对比设计。
- 如 WPF 原生毛玻璃实现成本过高，首版可采用“半透明面板 + 柔和阴影 + 模糊背景近似效果”。

## 8. 待确认视觉细节

- 是否支持浅色和深色主题。
- 是否显示系统托盘图标。
- 是否使用窗口置顶展示运行状态。
