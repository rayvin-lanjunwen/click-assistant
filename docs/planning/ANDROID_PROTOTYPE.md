# Click Assistant Android 端低保真原型

文档版本：v0.1.0
最后更新：2026-07-09
维护人：项目负责人
当前状态：初稿，用于 Android 可行性原型

本文档用于描述 Click Assistant Android 端第一阶段原型的页面结构和核心交互。原型目标是验证辅助功能授权、固定坐标点击、停止入口和本地任务保存，不追求完整视觉设计。

## 1. 页面原则

- 首屏完成主要流程，避免多层页面阻碍验证。
- 授权状态、任务配置、开始执行、停止执行和最近状态必须同时可见。
- 高风险动作必须有明显提示。
- 文案默认使用中文。

│ 任务名称                       │
│ [固定坐标点击测试任务        ] │
│                               │
│ 点击坐标                       │
│ X [ 500 ]    Y [ 900 ]        │
│                               │

│ [ 3 ]                         │
│                               │
│ 开始延迟                       │
│ [ 1000 ] ms                   │
│                               │

## 3. 授权引导

用户点击“打开辅助功能设置”后，系统跳转到 Android 辅助功能设置页。

```text
┌──────────────────────────────┐
│ Click Assistant 移动原型       │
│ 辅助功能：未启用               │
│                               │
│ 请在系统设置中启用             │
│ “Click Assistant 移动原型”。   │
│                               │
│ 该服务只用于执行你主动配置并    │
│ 主动启动的点击任务。            │
│                               │
│ [打开辅助功能设置]             │
└──────────────────────────────┘
```

授权后的首屏状态：

```text
┌──────────────────────────────┐
│ Click Assistant 移动原型       │
│ 辅助功能：已启用               │
│ [打开辅助功能设置]             │
└──────────────────────────────┘
```

## 4. 未授权执行提示

当用户未启用辅助功能服务时点击“开始执行”，原型应提示：

```text
┌──────────────────────────────┐
│ 无法执行                       │
│                               │
│ 请先开启辅助功能服务，          │
│ 再启动固定坐标点击任务。        │
│                               │
│ [打开辅助功能设置] [知道了]     │
└──────────────────────────────┘
```

第一阶段工程可以使用 Toast 或状态文本代替弹窗，但文案含义应保持一致。

## 5. 保存任务流程

```text
用户修改任务名称、坐标、重复次数、延迟和间隔
  ↓
点击“保存任务”
  ↓
校验输入
  ↓
写入本地存储
  ↓
最近状态显示“任务已保存”
```

校验规则：

- 任务名称不能为空。
- 坐标 `x` 和 `y` 必须大于或等于 `0`。
- 重复次数必须大于 `0`。
- 开始延迟和点击间隔必须大于或等于 `0`。

## 6. 执行流程

```text
用户点击“开始执行”
  ↓
校验辅助功能服务是否可用
  ↓
校验任务参数
  ↓
保存当前任务
  ↓
辅助功能服务按配置执行固定坐标点击
  ↓
最近状态更新为“执行中”
  ↓
执行结束后更新为“已完成”或“执行失败”
```

第一阶段暂不加入执行前确认页，避免扩大原型范围。产品化阶段应补充执行前安全确认。

## 7. 停止流程

```text
任务执行中
  ↓
用户点击“停止”
  ↓
辅助功能服务收到停止请求
  ↓
取消后续点击
  ↓
最近状态更新为“已停止”
```

停止按钮必须在首屏可见，不依赖系统返回键、通知栏或悬浮窗。

## 8. 最近状态

最近状态区域用于减少调试成本。

推荐状态：

- `未执行`
- `任务已保存`
- `执行中`
- `已完成`
- `已停止`
- `执行失败：辅助功能服务未启用`
- `执行失败：坐标无效`

## 9. 后续页面

第一阶段不做以下页面，但后续产品化可按顺序增加：

1. 任务列表页。
2. 多步骤编辑页。
3. 坐标采集辅助页。
4. 执行前确认页。
5. 执行日志页。
6. 权限和安全说明页。
